# Angular 2 DI ************************************************************************************************* * * * .----------------. .----------------. .----------------. * * | Injector +---------------->| Provider +----------------> | Dependency | * * | (DI Container) | hat viele | (Bauanleitung) | erzeugt | (Objekt) | * * '----------------' '----------------' '----------------' * ************************************************************************************************* ## Injector * ist der neue DI Container von Angular2 und stellt die zentrale API zur Verfügung * instanziiert Klassen "lazily", d.h. erst wenn Sie benötigt werden und **cached** die erzeugten Objekte anschließend * durch das Caching sind alle instanziierten Klassen Singletons ## Token * identifiziert eine Abhängigkeit * ist ein einfacher String oder ein Typ ## Provider * eine Bauanleitung / "Rezeptur" (recipe) um Tokens zu mappen: * auf einfache Werte * auf zu instanziierende Klassen * auf Factory Funktionen ## Dependency * beliege Abhängigkeiten (einfache Werte oder instanziierte Klassen) ## Quellen * http://blog.thoughtram.io/angular/2015/05/18/dependency-injection-in-angular-2.html * https://github.com/angular/angular/blob/master/modules/angular2/docs/di/di.md * https://angular.io/docs/ts/latest/api/core/Injector-class.html * https://angular.io/docs/js/latest/api/core/Injector-class.html
